# 基于 LLM 终端智能体（Agent）的电商全链路业务环比异动与负面评论全自动诊断系统

## 1. 项目背景与业务痛点 (Business Background)
在跨境电商（以巴西最大零售平台 Olist 为例）的日常经营中，面对 **11.4万笔全链路星型数仓数据** 与数十万条非结构化多语种用户评论，传统数据分析链路存在严重断层：
- 传统大盘 BI 仅能通过 Pandas/SQL 矩阵聚合下钻到“结果指标”暴跌（例如大盘 GMV 突发月环比暴跌）。
- 无法秒级渗透入底层非结构化文本的因果链（到底用户在骂什么？是物流瘫痪、商家缺件还是商品破损？）。
- 跨语言（葡萄牙语）的情感归因极度依赖人工翻译和文本聚类，策略产出耗时通常长达 **3天以上**。

为了实现真正的智能化降维提效，本项目基于 **Claude Code CLI 终端智能体环境** 托管本地雪花数仓，设计并打通了一条**“原始脏数据清洗 $\rightarrow$ 时序异动探针 $\rightarrow$ LLM 溢出鲁棒性防御 $\rightarrow$ 高管级商业策略自动交付”** 的端到端真·全自动 BI 数据管道（Pipeline）。

## 3. 工业级数据清洗与底池守卫 (Data Pipeline Defense)
脏数据是算法模型的死敌，在 Pipeline 运行前，脚本强制执行了预处理黄金预检防线：

- **DtypeWarning 强行拦截**：在 pd.read_csv 阶段，针对多表联结时由于空值频繁爆发的 Pandas 混合类型冲突地雷，强制指定 keep_default_na=False 与 low_memory=False 规避类型混淆。

- **底池真空过滤**：前置剔除 order_id 或 product_id 为空的残缺异常行，通过 how='left' 联结多表，守住流失率与大盘变动的分母底池，防止数据膨胀与指标失真。

- **文本脱水去噪**：针对捞出的低分负面评论，通过正则过滤大段纯符号或留空文本，为后续喂给 LLM 进行高效 Token 脱水。

## 4. 生产环境 LLM 鲁棒性防御攻坚 (LLMOps Robustness)
在多 Agent 协同的终端测试中，系统深入克服并解决了推理模型（Reasoning Models）在生产落地的两大底层技术死穴，完成了大厂级代码重构：

### 4.1 思考链配额截断（Thinking Token Exhaustion）
**事故现象**：调用 DeepSeek 满血版进行舆情观点聚类时，由于模型花费大量 Token 进行 <think>（深度思考），恶性侵占了单次 max_tokens（4096）总池配额，导致最后的 Valid JSON 输出写到一半被系统硬截断，引发底层 Python JSONDecodeError 当场死锁崩溃。

**自愈重构**：将接口升级适配，重构 API 调用算子并将输出配额强行拉满扩容至 8192 的硬件上限；同时在 Prompt 中进行结构化极简约束，压缩无意义思考路由，成功完成格式闭环。

### 4.2 大模型文本污染（Markdown Parsing Pollution）
**事故现象**：大语言模型常由于随机性，在返回 JSON 时贴心地包裹 ```json ... ``` 标记，或者带有“以下是分析结果：”等中文前缀。

**自愈重构**：在 Pipeline 中封装了带有正则表达式锚定技术（RegEx Anchoring）的解析函数 extract_json_from_response。运用 re.search(r'(\{.*\})', text, re.DOTALL)，无论外层包裹多少 markdown 噪音，皆可强行抠出最内层 Valid JSON 纯净大括号内容，大幅提升了自动化流的抗高并发波动的系统鲁棒性。

- 战术止血：系统自动推荐召回发券策略，针对受损用户精准包邮补发。

**场（次导因，占比 ~20%）· 物流配送严重延迟**：大促爆仓引发当地跨境物流雪崩。

- 战术止血：动态下调未来大促期间的动态 ETA 预期，接入第三方备用物流分流。

**货（微观因，占比 ~20%）· 商品质量低劣与破损**：部分低分评价集中在特定爆款商品商户。

- 战术止血：自动触发商户侧限流惩罚，启动退款运费险补贴。
